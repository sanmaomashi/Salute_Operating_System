# 系统信息

## 一、时间和日期

### 1. data

`date` 是一个常用的命令，用于显示或设置系统的日期和时间。

#### 1.1 显示当前日期和时间

```bash
$ date
```

上述命令将显示当前系统的日期和时间。

#### 1.2 设置日期和时间

```bash
$ date -s "YYYY-MM-DD HH:MM:SS"
```

使用上述命令，可以设置系统的日期和时间，其中 `YYYY-MM-DD` 是年月日，`HH:MM:SS` 是时分秒。

#### 1.3 常用选项

- `-u`：显示或设置为 UTC（协调世界时）。
- `+<格式>`：按指定的格式显示日期和时间。
- `-d "<日期/时间字符串>"`：显示指定日期或时间的格式化结果。

#### 1.4 示例

1. 显示当前日期和时间：

   ```bash
   $ date
   ```

2. 设置系统日期和时间为特定值：

   ```bash
   $ date -s "2023-05-17 10:30:00"
   ```

3. 显示当前日期和时间的 UTC（协调世界时）表示：

   ```bash
   $ date -u
   ```

4. 显示日期和时间的自定义格式：

   ```bash
   $ date +"%Y-%m-%d %H:%M:%S"
   ```

`date` 命令在系统管理、脚本编写和日志记录等方面非常有用。它提供了多种选项和格式化功能，可根据需要显示或设置日期和时间。

### 2. Cal

`cal`是一个常用的命令，用于显示日历。

#### 2.1 显示当前月份的日历

```bash
$ cal
```

上述命令将显示当前月份的日历，包含当前日期的突出显示。

#### 2.2 显示指定月份和年份的日历

```bash
$ cal <月份> <年份>
```

使用上述命令，可以显示指定月份和年份的日历。例如，`cal 5 2023`将显示2023年5月的日历。

#### 2.3 常用选项

- `-3`：显示当前月份及前后两个月的日历。
- `-y`：显示当前年份的整年日历。
- `-j`：以儒略历（Julian calendar）格式显示日历。

#### 2.4 示例

1. 显示当前月份的日历：

   ```bash
   $ cal
   ```

2. 显示指定月份和年份的日历：

   ```bash
   $ cal 5 2023
   ```

3. 显示当前月份及前后两个月的日历：

   ```bash
   $ cal -3
   ```

4. 显示当前年份的整年日历：

   ```bash
   $ cal -y
   ```

`cal`命令提供了一种简单的方式来查看日历，对于查看特定月份、计划活动或快速浏览日期非常有用。

## 二、磁盘信息

### 1. df

`df`是一个常用的命令，用于显示文件系统的磁盘空间使用情况。

#### 1.1 显示文件系统磁盘空间使用情况

```bash
$ df
```

上述命令将显示系统中所有文件系统的磁盘空间使用情况，包括文件系统的挂载点、总容量、已用空间、可用空间和使用百分比。

#### 1.2 常用选项

- `-h`：以人类可读的格式显示磁盘空间使用情况，使用适当的单位（如GB、MB）。
- `-T`：显示文件系统类型。
- `-i`：显示索引节点（inode）使用情况。

#### 1.3 示例

1. 显示文件系统的磁盘空间使用情况：

   ```bash
   $ df
   ```

2. 以人类可读的格式显示文件系统的磁盘空间使用情况：

   ```bash
   $ df -h
   ```

3. 显示文件系统类型、总容量和已用空间：

   ```bash
   $ df -T
   ```

4. 显示文件系统的索引节点（inode）使用情况：

   ```bash
   $ df -i
   ```

`df`命令对于监控文件系统的磁盘空间使用情况、查找可用空间或分析磁盘使用率非常有用。可以根据需要使用不同的选项来获取所需的磁盘空间信息。

### 2. du

`du`是一个常用的命令，用于计算目录或文件的磁盘使用量。

#### 2.1 显示目录或文件的磁盘使用量

```bash
$ du [选项] [目录/文件]
```

上述命令将显示指定目录或文件的磁盘使用量。

#### 2.2 常用选项

- `-h`：以人类可读的格式显示磁盘使用量，使用适当的单位（如GB、MB）。
- `-s`：仅显示总体磁盘使用量，不显示目录下每个子目录的详细使用量。
- `-c`：显示总体磁盘使用量，并在最后一行显示总计使用量。
- `-a`：显示目录中每个文件和子目录的磁盘使用量。
- `-d <深度>`：限制显示磁盘使用量的层级深度。

#### 2.3 示例

1. 显示目录的磁盘使用量：

   ```bash
   $ du /path/to/directory
   ```

2. 以人类可读的格式显示目录的磁盘使用量：

   ```bash
   $ du -h /path/to/directory
   ```

3. 显示文件的磁盘使用量：

   ```bash
   $ du /path/to/file
   ```

4. 显示目录的总体磁盘使用量：

   ```bash
   $ du -s /path/to/directory
   ```

5. 显示目录中每个文件和子目录的磁盘使用量：

   ```bash
   $ du -a /path/to/directory
   ```

`du`命令对于了解目录或文件占用的磁盘空间非常有用。可以根据需要使用不同的选项来获取所需的磁盘使用量信息。

### 3. free 

`free`是一个常用的命令，用于显示系统内存的使用情况。

#### 3.1 显示系统内存的使用情况

```bash
$ free
```

上述命令将显示系统中的内存使用情况，包括总内存、已用内存、可用内存、缓存和交换空间的使用情况。

#### 3.2 常用选项

- `-h`：以人类可读的格式显示内存使用情况，使用适当的单位（如GB、MB）。
- `-b`：以字节为单位显示内存使用情况。

#### 3.3 示例

1. 显示系统内存的使用情况：

   ```bash
   $ free
   ```

2. 以人类可读的格式显示系统内存的使用情况：

   ```bash
   $ free -h
   ```

3. 以字节为单位显示系统内存的使用情况：

   ```bash
   $ free -b
   ```

`free`命令对于监控系统内存的使用情况、查找可用内存或分析内存使用率非常有用。可以根据需要使用不同的选项来获取所需的内存信息。

## 三、进程信息

所谓进程，通俗地说就是当前正在执行的一个程序

### 1. ps

`ps`是一个常用的命令，用于查看当前正在运行的进程信息。

#### 1.1 显示当前正在运行的进程信息

```bash
$ ps
```

上述命令将显示当前用户的相关进程信息，包括进程ID（PID）、终端、CPU使用情况、内存使用情况和进程状态等。

#### 1.2 常用选项

- `-e`：显示所有进程，而不仅限于当前用户。
- `-f`：显示完整的进程信息，包括父进程ID（PPID）、CPU占用率等。
- `-l`：以长格式显示进程信息，包括进程的命令行参数。
- `-u`：显示与指定用户相关的进程信息。

#### 1.3 示例

1. 显示当前用户的进程信息：

   ```bash
   $ ps
   ```

2. 显示所有进程的完整信息：

   ```bash
   $ ps -ef
   ```

3. 以长格式显示当前用户的进程信息：

   ```bash
   $ ps -l
   ```

4. 显示指定用户的进程信息：

   ```bash
   $ ps -u username
   ```

`ps`命令对于查看系统中正在运行的进程、检查进程状态、定位资源占用等非常有用。可以根据需要使用不同的选项来获取所需的进程信息。

### 2. top

`top`是一个交互式的命令行工具，用于实时监视系统的进程活动和系统性能。

#### 2.1 执行top命令

```bash
$ top
```

上述命令将打开一个交互式的top界面，显示当前系统的进程列表和系统性能信息。默认情况下，进程列表按CPU使用率降序排列。

#### 2.2 top界面的一些常用命令

- `q`：退出top命令。
- `k`：终止选定的进程。
- `r`：修改选定的进程的优先级。
- `H`：显示线程视图，展示每个进程的线程信息。
- `1`：切换到全局CPU使用情况视图。
- `m`：切换内存使用情况视图。
- `t`：切换到显示每个进程的CPU时间的视图。
- `P`：按CPU使用率进行排序。
- `M`：按内存使用率进行排序。

除了交互界面外，还可以通过使用top的命令行选项来执行非交互式的操作，例如：

```bash
$ top -b -n 1
```

上述命令将以非交互式模式执行top，并显示一次系统的进程和性能信息。

`top`命令对于实时监视系统的进程活动、CPU使用率、内存使用率等非常有用。它提供了丰富的功能和交互性，可以方便地查看和管理系统资源。

### 3. kill

`kill`命令用于终止正在运行的进程。

#### 3.1 终止进程：

```bash
$ kill [选项] 进程ID
```

上述命令将向指定进程发送终止信号，以终止该进程的执行。

#### 3.2 常用选项

- ```bash
  -s <信号>
  ```

  ：指定要发送的信号。常见的信号包括：

  - `SIGTERM`（默认）：终止进程。
  - `SIGKILL`：强制终止进程，不允许进程进行清理操作。

- `-l`：列出所有可用的信号。

#### 3.3 示例

1. 终止进程：

   ```bash
   $ kill 1234
   ```

2. 发送指定信号终止进程：

   ```bash
   $ kill -s SIGTERM 1234
   ```

3. 列出所有可用的信号：

   ```bash
   $ kill -l
   ```

请注意，在终止进程之前，请确保使用正确的进程ID，并谨慎使用`SIGKILL`信号，因为它会强制终止进程，可能导致数据丢失或不一致的状态。

`kill`命令对于终止不响应或不需要继续运行的进程非常有用。可以使用进程管理工具（如`ps`）来获取进程ID，然后使用`kill`命令终止指定的进程。

## 四、日志信息

### 1. tail

`tail`是一个常用的命令，用于显示文件的末尾内容。

#### 1.1 显示文件末尾的内容

```bash
$ tail [选项] 文件
```

上述命令将显示指定文件的末尾内容，默认情况下显示文件的最后10行。

#### 1.2 常用选项

- `-n <行数>`：显示文件末尾指定行数的内容。
- `-f`：跟踪文件的变化，实时显示新增的内容。
- `-c <字节数>`：显示文件末尾指定字节数的内容。
- `-q`：静默模式，禁止显示文件名。
- `-s <秒数>`：与`-f`选项一起使用，设置刷新间隔时间。

#### 1.3 示例

1. 显示文件的末尾10行：

   ```bash
   $ tail file.txt
   ```

2. 显示文件的末尾20行：

   ```bash
   $ tail -n 20 file.txt
   ```

3. 实时显示文件的新增内容：

   ```bash
   $ tail -f file.txt
   ```

4. 显示文件的末尾100字节：

   ```bash
   $ tail -c 100 file.txt
   ```

`tail`命令对于查看日志文件的最新记录、实时监视文件变化等非常有用。根据需要使用不同的选项来获取所需的文件末尾内容。
